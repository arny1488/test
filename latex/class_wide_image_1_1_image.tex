Resize the image to given dimensions.

\$width and \$height are both smart coordinates. This means that you can pass any of these values in\+:
\begin{DoxyItemize}
\item positive or negative integer (100, -\/20, ...)
\item positive or negative percent string (30\%, -\/15\%, ...)
\item complex coordinate (50\% -\/ 20, 15 + 30\%, ...)
\item null\+: if one dimension is null, it\textquotesingle{}s calculated proportionally from the other.
\end{DoxyItemize}

\$fit parameter can be set to one of these three values\+:
\begin{DoxyItemize}
\item \textquotesingle{}inside\textquotesingle{}\+: resize proportionally and fit the resulting image tightly in the \$width x \$height box
\item \textquotesingle{}outside\textquotesingle{}\+: resize proportionally and fit the resulting image tighly outside the box
\item \textquotesingle{}fill\textquotesingle{}\+: resize the image to fill the \$width x \$height box exactly
\end{DoxyItemize}

\$scale parameter can be\+:
\begin{DoxyItemize}
\item \textquotesingle{}down\textquotesingle{}\+: only resize the image if it\textquotesingle{}s larger than the \$width x \$height box
\item \textquotesingle{}up\textquotesingle{}\+: only resize the image if it\textquotesingle{}s smaller than the \$width x \$height box
\item \textquotesingle{}any\textquotesingle{}\+: resize the image

resize\+Canvas (\begin{DoxyParamCaption}\item[{}]{\$width}{, }\item[{}]{\$height}{, }\item[{}]{\$pos\+\_\+x}{, }\item[{}]{\$pos\+\_\+y}{, }\item[{}]{\$bg\+\_\+color}{ = {\ttfamily null}, }\item[{}]{\$scale}{ = {\ttfamily \textquotesingle{}any\textquotesingle{}}, }\item[{}]{\$merge}{ = {\ttfamily false}}\end{DoxyParamCaption})}

Resizes the canvas of the image, but doesn\textquotesingle{}t scale the content of the image

This operation creates an empty canvas with dimensions \$width x \$height, filled with background color \$bg\+\_\+color and draws the original image onto it at position \mbox{[}\$pos\+\_\+x, \$pos\+\_\+y\mbox{]}.

Arguments \$width, \$height, \$pos\+\_\+x and \$pos\+\_\+y are all smart coordinates. \$width and \$height are relative to the current image size, \$pos\+\_\+x and \$pos\+\_\+y are relative to the newly calculated canvas size. This can be confusing, but it makes sense. See the example below.

The example below loads a 100x150 image and then resizes its canvas to 200\% x 100\%+20 (which evaluates to 200x170). The image is placed at position \mbox{[}10, center+20\mbox{]}, which evaluates to \mbox{[}10, 30\mbox{]}. {\ttfamily  \$image = \doxylink{class_wide_image_1_1_wide_image_ac0d79fb0b00418ed15fbe8f6047ff4b2}{Wide\+Image\textbackslash{}load}(\textquotesingle{}someimage.\+jpg\textquotesingle{}); // 100x150 \$white = \$image-\/\texorpdfstring{$>$}{>}allocate\+Color(255, 255, 255); \$image-\/\texorpdfstring{$>$}{>}resize\+Canvas(\textquotesingle{}200\textquotesingle{}, \textquotesingle{}100\% + 20\textquotesingle{}, 10, \textquotesingle{}center+20\textquotesingle{}, \$white); }

The parameter \$merge defines whether the original image should be merged onto the new canvas. This means it blends transparent color and alpha colors into the background color. If set to false, the original image is just copied over, preserving the transparency/alpha information.

You can set the \$scale parameter to limit when to resize the canvas. For example, if you want to resize the canvas only if the image is smaller than the new size, but leave the image intact if it\textquotesingle{}s larger, set it to \textquotesingle{}up\textquotesingle{}. Likewise, if you want to shrink the canvas, but don\textquotesingle{}t want to change images that are already smaller, set it to \textquotesingle{}down\textquotesingle{}.
